# /i18n — Internationalization

> Sử dụng khi cần thêm/sửa translation keys, kiểm tra hardcoded text, hoặc thêm ngôn ngữ mới.

// turbo-all

---

## Bước 1: Scan Hardcoded Text

1. Tìm hardcoded text trong frontend:
   ```
   Tìm trong packages/app/features/ và packages/ui/:
   - Text tiếng Việt hardcoded (không qua t())
   - Text tiếng Anh hardcoded
   - Placeholder text, error messages, labels hardcoded
   - Title, description, button text hardcoded
   ```

2. Dùng `grep_search` để tìm patterns:
   - Tìm strings tiếng Việt (Unicode characters)
   - Tìm JSX text content không dùng `t()`
   - Kiểm tra `aria-label`, `placeholder`, `title` attributes

3. Tạo danh sách tất cả text cần i18n

## Bước 2: Thêm i18n Keys

### Naming Convention
```
{module}.{page}.{element}

Ví dụ:
federation.regulations.title         → "Quy chế" / "Regulations"
federation.regulations.description   → "Quản lý quy chế..." / "Manage regulations..."
athlete.profile.belt_history         → "Lịch sử đai" / "Belt History"
common.actions.save                  → "Lưu" / "Save"
common.actions.cancel                → "Huỷ" / "Cancel"
common.actions.delete                → "Xoá" / "Delete"
common.errors.network                → "Lỗi kết nối" / "Network error"
```

### Phân loại Keys
| Prefix | Loại | Scope |
|--------|------|-------|
| `common.*` | Actions, errors, labels dùng chung | Toàn app |
| `{module}.*` | Keys dành riêng cho module | Module-specific |
| `nav.*` | Sidebar, breadcrumb, menu labels | Navigation |
| `auth.*` | Login, register, logout messages | Authentication |
| `validation.*` | Form validation messages | Forms |

### Thêm keys cho CẢ 2 ngôn ngữ
Mỗi key phải có bản dịch cho:
- 🇻🇳 **vi** (Vietnamese) — ngôn ngữ chính
- 🇬🇧 **en** (English) — ngôn ngữ phụ

## Bước 3: Thay Thế Text Trong Code

### Trong Components
```tsx
// ❌ SAI — Hardcoded text
<h1>Quản lý quy chế</h1>
<button>Lưu thay đổi</button>

// ✅ ĐÚNG — Dùng i18n
const { t } = useI18n()
<h1>{t('federation.regulations.title')}</h1>
<button>{t('common.actions.save')}</button>
```

### Attributes
```tsx
// ❌ SAI
<input placeholder="Nhập tên..." />

// ✅ ĐÚNG
<input placeholder={t('common.form.name_placeholder')} />
```

### Error Messages
```tsx
// ❌ SAI
setError('Không tìm thấy dữ liệu')

// ✅ ĐÚNG
setError(t('common.errors.not_found'))
```

### Dynamic Content
```tsx
// Với biến interpolation
t('tournament.match.score_display', { score: 10, max: 15 })
// → "Điểm: 10/15" hoặc "Score: 10/15"
```

## Bước 4: Verify

1. **Kiểm tra TypeScript**:
   ```bash
   npm run typecheck
   ```

2. **Kiểm tra tiếng Việt**:
   - Mở app với locale = vi
   - Kiểm tra tất cả trang hiển thị tiếng Việt đúng
   - Không có key nào hiển thị raw (e.g., `federation.title` thay vì "Liên đoàn")

3. **Kiểm tra tiếng Anh**:
   - Chuyển locale = en
   - Kiểm tra tất cả trang hiển thị tiếng Anh đúng
   - Không có text tiếng Việt lọt vào bản English

4. **Checklist cuối**:
   - [ ] Không còn hardcoded text trong code
   - [ ] Mọi key có bản dịch vi + en
   - [ ] Không có missing translation warnings
   - [ ] UI hiển thị đúng cho cả 2 ngôn ngữ
   - [ ] Tên keys follow naming convention
   - [ ] TypeScript build pass
